prototype-optics

Build browser prototypes that can't drift off the RoleModel Optics design system.

Three deterministic PreToolUse hooks run on every Write/Edit. Anything off-system is hard-blocked (exit 2) and the reason is fed back to the agent to fix — so a non-compliant file can't be saved.

The three guards

Value
Colors & sizes must be var(--op-*) tokens. No raw #hex, rgb(), or 12px — anywhere.
Classname
Every class must be a real Optics class, a configured prefix, or an allow-list entry. Typos are rejected with suggestions.
BEM
A BEM element X__Y must sit inside its block X — e.g. card__header only inside .card.

The three modes

Mode Classes & values Custom prefixes Token redefinition
optics-only Pure Optics or fail No No
prefixed default Optics + your prefix as names; values stay pure Optics Yes (names only) No
theme Everything in prefixed Yes Seed tokens only (H/S, fonts, letter-spacing, input-height)

Examples

Allowed
.cta { color: var(--op-color-primary-base); padding: var(--op-space-medium); }
<button class="btn btn--primary">Go</button>
Blocked
.cta { color: #2f6fed; }   /* raw color -> use var(--op-color-*) */
.cta { padding: 12px; }    /* raw length -> token or calc(N * var(--op-size-unit)) */
<div class="megamenu">     /* not an Optics class */
theme mode only
:root { --op-color-primary-h: 265; }   /* re-skin the whole scale from one seed */

Set the mode in .claude/optics-guard.json — { "mode": "prefixed" }. Pairs with the optics-context and bem-structure skills.
